# Contributing to Hammer-Hub

First off, thank you for considering contributing to Hammer-Hub. It's people like you that make Hammer-Hub such a great tool.

Following these guidelines helps to communicate that you respect the time of the developers managing and developing this open source project. In return, they should reciprocate that respect in addressing your issue, assessing changes, and helping you finalize your pull requests.

## Getting Started

Before you begin:
- Make sure you have a [GitHub account](https://github.com/signup/free)
- Check if your issue [has already been reported](https://github.com/lisiCAO/hammer-hub/issues).
- Fork the repository and create your branch from `main`.

## Making Changes

- Write clear and meaningful Git commit messages.
- Make sure your code lints (if applicable).
- Update the README.md with details of changes to the interface, this includes new environment variables, exposed ports, useful file locations and container parameters.
- Add or update the tests if applicable.
- Add or update the docs when necessary.

## Submitting Changes

1. Push your changes to a branch in your fork of the repository.
2. Submit a pull request to the original repository.
3. Direct the pull request to the `development` branch, if there is one.
4. Include a description of what the changes do and mention any [issues](https://github.com/lisiCAO/hammer-hub/issues) your pull request is addressing.

## Code Review Process

The core team looks at Pull Requests on a regular basis in a weekly triage meeting. After feedback has been given, we expect responses within two weeks. After two weeks, we may close the pull request if it isn't showing any activity.

## Code of Conduct

Our Code of Conduct means we expect all contributors to treat each other with respect and courtesy. Harassment and abuse are strictly forbidden.

## Recognition model

Contributors who show dedication and skill are recognized through a system of rewards. This might include shout-outs in community channels, priority for future proposals, and so on.

## Questions

If you have any questions about contributing, please feel free to contact us.
